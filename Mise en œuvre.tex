\chapter{Mise en œuvre}

\section{Introduction}
Ce chapitre présente la mise en œuvre concrète de la fonctionnalité 
de recueil de feedbacks et de réclamations développée dans le cadre 
du stage chez Bewize. Après avoir défini les besoins dans le chapitre 
de cadrage et établi la conception dans le chapitre précédent, il 
s'agit maintenant de décrire en détail l'implémentation technique 
de chaque composant de la solution.

Ce chapitre suit la logique de développement adoptée pendant le stage, 
en présentant successivement : les technologies utilisées, la mise en 
place de l'environnement de développement, le développement du formulaire 
WebView en React JS, l'intégration dans l'application mobile React Native, 
le développement du back-end Spring Boot, la mise en place de la base 
de données PostgreSQL, et enfin le développement du dashboard monitor.

\subsection{Introduction aux choix technologiques}
Le développement de la fonctionnalité de recueil de feedbacks et de 
réclamations chez Bewize repose sur un ensemble de technologies modernes, 
complémentaires et éprouvées. Le choix de ces technologies a été guidé 
par les contraintes de l'environnement existant chez Bewize, les bonnes 
pratiques du développement logiciel moderne, et les exigences de 
performance et de maintenabilité du projet. Chaque technologie a été 
sélectionnée pour son adéquation avec les besoins spécifiques du projet 
et sa complémentarité avec les autres technologies de la stack technique.

\subsection{React JS}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.22\textwidth]{Figures/reactjs.png}
    \caption{Logo React JS}
    \label{fig:reactjs}
\end{figure}

React JS est une bibliothèque JavaScript open-source développée et 
maintenue par Meta (anciennement Facebook), utilisée pour construire 
des interfaces utilisateur web interactives, réactives et performantes. 
Publiée en 2013, React JS a révolutionné le développement front-end 
grâce à son approche innovante basée sur les composants réutilisables 
et le DOM virtuel (Virtual DOM).

Le principe fondamental de React JS repose sur la décomposition de 
l'interface utilisateur en composants indépendants, réutilisables et 
auto-suffisants. Chaque composant gère son propre état interne et peut 
recevoir des données externes via des propriétés appelées props. Cette 
architecture componentisée facilite considérablement la maintenance, 
les tests et l'évolution de l'interface au fil du temps.

\textbf{Utilisation dans le projet :}
\begin{itemize}
    \item \textbf{La page WebView :} formulaire de saisie de feedback 
    ou réclamation, accessible depuis l'application mobile via le 
    composant WebView de React Native
    \item \textbf{Le dashboard monitor :} interface interne de 
    consultation et traitement des retours utilisateurs, accessible 
    par l'équipe Bewize
\end{itemize}

\textbf{Avantages pour le projet :}
\begin{itemize}
    \item Développement rapide grâce aux composants réutilisables
    \item Interface réactive et fluide grâce au DOM virtuel
    \item Intégration simple avec les APIs REST Spring Boot
    \item Responsive design optimal pour l'affichage mobile via WebView
\end{itemize}

\section{Développement du formulaire WebView}

\subsection{Création du projet React JS}
La première étape du développement a consisté à créer le projet React JS 
qui hébergera le formulaire WebView. Ce projet a été initialisé en 
utilisant Create React App, l'outil officiel de création de projets 
React JS, qui configure automatiquement un environnement de développement 
complet avec webpack, Babel et les outils de test.

La structure du projet React JS a été organisée de manière claire et 
maintenable, en séparant les composants, les services et les styles :

\begin{itemize}
    \item \textbf{src/components/ :} composants React JS réutilisables
    \item \textbf{src/services/ :} services de communication avec l'API
    \item \textbf{src/styles/ :} fichiers CSS et styles globaux
    \item \textbf{src/utils/ :} fonctions utilitaires partagées
\end{itemize}

\subsection{Développement du composant formulaire}
Le composant principal du formulaire WebView a été développé en React JS. 
Il permet à l'utilisateur de saisir son feedback ou sa réclamation 
de manière simple et intuitive. Le formulaire comprend les champs suivants :

\begin{itemize}
    \item \textbf{Type de retour :} un sélecteur permettant de choisir 
    entre "Feedback" et "Réclamation"
    \item \textbf{Message :} une zone de texte permettant à l'utilisateur 
    de saisir son message de manière détaillée
    \item \textbf{Bouton d'envoi :} un bouton clairement identifiable 
    permettant de soumettre le formulaire
\end{itemize}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.6\textwidth]{Figures/webview_form.png}
    \caption{Interface du formulaire WebView}
    \label{fig:webview_form}
\end{figure}

\subsection{Validation des données du formulaire}
Une validation côté client a été implémentée pour s'assurer que les 
données saisies par l'utilisateur sont correctes avant leur envoi 
au back-end. Cette validation vérifie :

\begin{itemize}
    \item La présence obligatoire du type de retour (feedback ou réclamation)
    \item La présence obligatoire du message avec une longueur minimale 
    de dix caractères
    \item La longueur maximale du message pour éviter les abus
\end{itemize}

En cas de données invalides, des messages d'erreur clairs et précis 
sont affichés directement sous les champs concernés, guidant l'utilisateur 
vers une saisie correcte.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.6\textwidth]{Figures/webview_validation.png}
    \caption{Validation des champs du formulaire}
    \label{fig:webview_validation}
\end{figure}

\subsection{Connexion à l'API Spring Boot}
Une fois le formulaire validé, les données sont envoyées au back-end 
Spring Boot via une requête HTTP POST. Cette communication est gérée 
par un service dédié qui utilise l'API Fetch native de JavaScript 
pour effectuer les appels HTTP de manière asynchrone.

La réponse du back-end est traitée de manière appropriée :
\begin{itemize}
    \item En cas de succès (HTTP 200) : un message de confirmation 
    est affiché à l'utilisateur et le formulaire est réinitialisé
    \item En cas d'erreur réseau : un message d'erreur explicite 
    est affiché invitant l'utilisateur à réessayer ultérieurement
    \item En cas d'erreur serveur (HTTP 500) : un message d'erreur 
    générique est affiché
\end{itemize}

\begin{figure}[H]

    \includegraphics[width=0.6\textwidth]{Figures/webview_confirmation.png}

\section{Développement du back-end Spring Boot}
%==========================================================

\subsection{Création du projet Spring Boot}
Le projet Spring Boot a été créé en utilisant Spring Initializr, 
l'outil officiel de génération de projets Spring Boot. Les dépendances 
suivantes ont été sélectionnées lors de la création du projet :

\begin{itemize}
    \item \textbf{Spring Web :} pour la création des APIs REST
    \item \textbf{Spring Data JPA :} pour la persistance des données
    \item \textbf{PostgreSQL Driver :} pour la connexion à la base 
    de données PostgreSQL
    \item \textbf{Spring Validation :} pour la validation des données
    \item \textbf{Lombok :} pour la réduction du code boilerplate
    \item \textbf{SpringDoc OpenAPI :} pour la documentation 
    automatique des APIs via Swagger
\end{itemize}

\subsection{Développement de l'entité Feedback}
L'entité Feedback est la classe centrale du modèle de données. 
Elle est annotée avec les annotations JPA nécessaires pour assurer 
sa persistance dans la base de données PostgreSQL et contient 
les attributs suivants :

\begin{itemize}
    \item \textbf{id :} identifiant unique auto-généré de type Long
    \item \textbf{type :} énumération indiquant si c'est un FEEDBACK 
    ou une RECLAMATION
    \item \textbf{message :} contenu textuel du retour utilisateur
    \item \textbf{dateEnvoi :} date et heure d'envoi générées 
    automatiquement à la création
    \item \textbf{statut :} énumération indiquant le statut de 
    traitement (NOUVEAU, EN\_COURS, TRAITE)
    \item \textbf{utilisateurId :} identifiant de l'utilisateur 
    ayant soumis le retour
\end{itemize}

\subsection{Développement du Repository}
Le FeedbackRepository est une interface Spring Data JPA qui étend 
JpaRepository. Elle fournit automatiquement toutes les méthodes CRUD 
standard (save, findById, findAll, delete) sans nécessiter d'implémentation 
manuelle. Des méthodes de requête personnalisées ont également été 
définies pour les besoins spécifiques du dashboard monitor :

\begin{itemize}
    \item Recherche par type (FEEDBACK ou RECLAMATION)
    \item Recherche par statut de traitement
    \item Recherche avec filtres combinés (type et statut)
    \item Comptage par type pour les statistiques
\end{itemize}

\subsection{Développement du Service}
Le FeedbackService implémente la logique métier de l'application. 
Il constitue la couche intermédiaire entre le contrôleur et le 
repository, garantissant une séparation claire des responsabilités :

\begin{itemize}
    \item \textbf{Création d'un feedback :} validation des données 
    reçues, initialisation du statut à NOUVEAU et de la date d'envoi, 
    puis sauvegarde via le repository
    \item \textbf{Récupération des feedbacks :} application des filtres 
    demandés par le dashboard et retour des résultats
    \item \textbf{Mise à jour du statut :} vérification de la validité 
    du nouveau statut et mise à jour en base de données
    \item \textbf{Statistiques :} calcul des comptages par type et 
    par statut pour le tableau de bord
\end{itemize}

Le FeedbackController expose les endpoints REST API consommés par 
la WebView et le dashboard monitor. Il est annoté avec 
\texttt{@RestController} et \texttt{@CrossOrigin} pour permettre

\subsection{Gestion des erreurs}
Un gestionnaire d'exceptions global a été mis en place via 
\texttt{@ControllerAdvice} pour intercepter et traiter de manière 
uniforme toutes les exceptions levées par l'application :

\begin{itemize}
    \item \textbf{ResourceNotFoundException :} retourne HTTP 404 
    lorsqu'un feedback ou réclamation n'est pas trouvé
    \item \textbf{ValidationException :} retourne HTTP 400 avec 
    les détails des erreurs de validation
    \item \textbf{Exception générale :} retourne HTTP 500 avec 
    un message d'erreur générique
\end{itemize}

    de la page
    \item Permettre le retour à l'écran précédent après l'envoi

\subsection{Présentation du dashboard}
Le dashboard monitor est une interface web interne développée en 
React JS, accessible exclusivement par l'équipe interne de Bewize. 
Il constitue le point central de consultation et de traitement 
des feedbacks et réclamations reçus des utilisateurs de l'application

\subsection{Page liste des retours}
La page principale du dashboard affiche la liste complète de tous 
les feedbacks et réclamations reçus, sous forme de tableau paginé. 
Chaque ligne du tableau affiche les informations essentielles :

\begin{itemize}
    \item L'identifiant unique du retour
    \item Le type (feedback ou réclamation) avec un badge coloré distinctif
    \item Un extrait du message pour une lecture rapide
    \item La date d'envoi formatée
    \item Le statut de traitement avec un indicateur visuel coloré
    \item Des boutons d'action pour consulter le détail ou changer 
    le statut directement depuis la liste
\end{itemize}

plusieurs critères :

\begin{itemize}

    ou uniquement les réclamations
    \item \textbf{Filtre par statut :} afficher uniquement les retours 
    NOUVEAU, EN\_COURS ou TRAITE
    \item \textbf{Filtre par date :} sélectionner une plage de dates 
    pour afficher les retours d'une période spécifique
    \item \textbf{Recherche textuelle :} rechercher dans le contenu

\section{Conclusion}
%==========================================================

Ce chapitre a présenté en détail la mise en œuvre complète de la 
fonctionnalité de recueil de feedbacks et de réclamations développée 
dans le cadre du stage chez Bewize. L'implémentation a couvert 
l'ensemble des composants de la solution : les technologies utilisées, 
le formulaire WebView React JS, le back-end Spring Boot avec ses APIs 
REST, la base de données PostgreSQL et le dashboard monitor.

Chaque composant a été développé en suivant les bonnes pratiques 
du développement logiciel moderne, avec une attention particulière 
portée à la qualité du code, à la validation des données et à la 
gestion des erreurs. Les tests fonctionnels réalisés ont confirmé 
le bon fonctionnement du parcours complet, de la saisie du retour 
par l'utilisateur mobile jusqu'à sa consultation et son traitement 
par l'équipe interne Bewize dans le dashboard monitor.

Le chapitre suivant présentera les résultats obtenus et une évaluation 
globale de la solution développée.
