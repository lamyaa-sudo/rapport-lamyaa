\chapter{Mise en œuvre}

\section{Introduction}
Ce chapitre présente la mise en œuvre concrète de la fonctionnalité 
de recueil de feedbacks et de réclamations développée dans le cadre 
du stage chez Bewize. Après avoir défini les besoins dans le chapitre 
de cadrage et établi la conception dans le chapitre précédent, il 
s'agit maintenant de décrire en détail l'implémentation technique 
de chaque composant de la solution.

Ce chapitre suit la logique de développement adoptée pendant le stage, 
en présentant successivement : les technologies utilisées, la mise en 
place de l'environnement de développement, le développement du formulaire 
WebView en React JS, le développement du back-end Spring Boot, la mise 
en place de la base de données PostgreSQL, et enfin le développement 
du dashboard monitor.

\begin{itemize}

\section{Développement du back-end Spring Boot}
%==========================================================

\subsection{Création du projet Spring Boot}
Le projet Spring Boot a été créé avec les dépendances suivantes :

\begin{itemize}
    \item \textbf{Spring Web :} pour la création des APIs REST
    \item \textbf{Spring Data JPA :} pour la persistance des données
    \item \textbf{PostgreSQL Driver :} pour la connexion à PostgreSQL
    \item \textbf{Spring Validation :} pour la validation des données
    \item \textbf{Lombok :} pour la réduction du code boilerplate
    \item \textbf{SpringDoc OpenAPI :} pour la documentation Swagger
\end{itemize}

\subsection{Développement de l'entité Feedback}
L'entité Feedback est la classe centrale du modèle avec les attributs :

\begin{itemize}
    \item \textbf{id :} identifiant unique auto-généré
    \item \textbf{type :} FEEDBACK ou RECLAMATION
    \item \textbf{message :} contenu textuel du retour
    \item \textbf{dateEnvoi :} date et heure d'envoi
    \item \textbf{statut :} NOUVEAU, EN\_COURS ou TRAITE
    \item \textbf{utilisateurId :} identifiant de l'utilisateur
\end{itemize}

\subsection{Développement du Controller}
Le FeedbackController expose les endpoints REST API suivants :

\begin{table}[H]
\centering
\caption{Endpoints REST API implémentés dans le FeedbackController}
\label{tab:endpoints}
\renewcommand{\arraystretch}{1.8}
\begin{tabular}{|p{1.8cm}|p{5cm}|p{2cm}|p{4cm}|}

\textbf{Méthode} & \textbf{Endpoint} & \textbf{Code retour} & 
\textbf{Description} \\
\hline
POST & /api/feedbacks & 201 Created & Créer un nouveau retour \\
\hline
GET & /api/feedbacks & 200 OK & Liste de tous les retours \\
\hline
GET & /api/feedbacks/\{id\} & 200 OK & Détail d'un retour \\
\hline

\hline

\end{tabular}
\end{table}

Le tableau \ref{tab:endpoints} présente les cinq endpoints REST API 
développés dans le FeedbackController. L'endpoint POST retourne un 
code HTTP 201 lors de la création réussie d'un retour, tandis que 
les endpoints GET et PUT retournent un code HTTP 200.

\section{Conclusion}
%==========================================================

Ce chapitre a présenté en détail la mise en œuvre complète de la 
fonctionnalité de recueil de feedbacks et de réclamations développée 
dans le cadre du stage chez Bewize. L'implémentation a couvert 
l'ensemble des composants de la solution : les technologies utilisées, 
le formulaire WebView React JS, le back-end Spring Boot avec ses APIs 
REST, la base de données PostgreSQL et le dashboard monitor.

Chaque composant a été développé en suivant les bonnes pratiques du 
développement logiciel moderne, avec une attention particulière portée 
à la qualité du code, à la validation des données et à la gestion des 
erreurs. Le chapitre suivant présentera les résultats des tests 
réalisés et une évaluation globale de la solution développée.
