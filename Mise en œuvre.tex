\chapter{Mise en œuvre}

\section{Introduction}
Ce chapitre présente la mise en œuvre concrète de la fonctionnalité 
de recueil de réclamations développée dans le cadre 
du stage chez Bewize. Après avoir défini les besoins dans le chapitre 
de cadrage et établi la conception dans le chapitre précédent, il 
s'agit maintenant de décrire en détail l'implémentation technique 
de chaque composant de la solution.

Ce chapitre suit la logique de développement adoptée pendant le stage, 
en présentant successivement : les technologies utilisées, la mise en 
place de l'environnement de développement, le développement du formulaire 
WebView en React JS, le développement du back-end Spring Boot, la mise 
en place de la base de données PostgreSQL, et enfin le développement 
du dashboard monitor.

\section{Développement du formulaire WebView}
%==========================================================

\subsection{Création du projet React JS}
La première étape du développement a consisté à créer le projet React JS 
qui hébergera le formulaire WebView. La structure du projet est organisée 
comme suit :

\begin{itemize}
    \item \textbf{src/components/ :} composants React JS réutilisables
    \item \textbf{src/services/ :} services de communication avec l'API
    \item \textbf{src/styles/ :} fichiers CSS et styles globaux
    \item \textbf{src/utils/ :} fonctions utilitaires partagées
\end{itemize}

\subsection{Développement du composant formulaire}
Le composant principal du formulaire WebView comprend les champs suivants :

\begin{itemize}
    \item \textbf{Type de retour :} sélecteur entre "Feedback" et 
    "Réclamation"
    \item \textbf{Informations utilisateur :} prénom, nom, email, 
    numéro étudiant et statut
    \item \textbf{Détails de la réclamation :} type, objet, description 
    et pièce jointe
    \item \textbf{Bouton d'envoi :} bouton de soumission du formulaire
\end{itemize}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.6\textwidth]{Figures/webview_form_1.png}
    \includegraphics[width=0.6\textwidth]{Figures/webview_form_2.png}
    \caption{Interface complète du formulaire WebView de recueil de réclamations}
    \label{fig:webview_form}
\end{figure}

La figure \ref{fig:webview_form} présente l'interface complète du 
formulaire WebView développé en React JS. La partie supérieure affiche 
les informations personnelles de l'utilisateur, tandis que la partie 
inférieure présente les détails de la réclamation ainsi que le bouton 
d'envoi. L'interface est épurée, intuitive et adaptée à un affichage 
sur écran mobile.

\subsection{Validation des données du formulaire}
Une validation côté client a été implémentée pour vérifier :

\begin{itemize}
    \item La présence obligatoire de tous les champs requis
    \item La présence obligatoire du message avec une longueur minimale
    \item La longueur maximale du message
\end{itemize}

\begin{figure}[H]

    \includegraphics[width=0.6\textwidth]{Figures/webview_validation.png}

La figure \ref{fig:webview_validation} illustre le comportement du

Des messages d'erreur explicites sont affichés en rouge directement

\section{Développement du back-end Spring Boot}
%==========================================================

Le back-end Spring Boot a été développé en suivant une architecture 
en couches, comprenant les composants Controller, Service et Repository. 
Il expose des APIs REST pour la gestion et le suivi des réclamations 
des utilisateurs.

\subsection{Développement du Controller}
Le FeedbackController expose les endpoints REST API suivants :

\begin{table}[H]
\centering
\caption{Endpoints REST API implémentés dans le FeedbackController}
\label{tab:endpoints}
\renewcommand{\arraystretch}{1.8}
\begin{tabular}{|p{1.8cm}|p{5cm}|p{2cm}|p{4cm}|}
\hline
\textbf{Méthode} & \textbf{Endpoint} & \textbf{Code retour} & 
\textbf{Description} \\
\hline
POST & /api/feedbacks & 201 Created & Créer une nouvelle réclamation \\
\hline
GET & /api/feedbacks & 200 OK & Liste de toutes les réclamations \\
\hline
GET & /api/feedbacks/\{id\} & 200 OK & Détail d'une réclamation \\
\hline
PUT & /api/feedbacks/\{id\}/statut & 200 OK & Mettre à jour le statut \\
\hline
GET & /api/feedbacks/stats & 200 OK & Statistiques des réclamations \\
\hline
\end{tabular}
\end{table}

Le tableau \ref{tab:endpoints} présente les cinq endpoints REST API 
développés dans le FeedbackController. L'endpoint POST retourne un 
code HTTP 201 lors de la création réussie d'une réclamation, tandis que 
les endpoints GET et PUT retournent un code HTTP 200.

\section{Conclusion}
%==========================================================

Ce chapitre a présenté en détail la mise en œuvre complète de la 
fonctionnalité de recueil de réclamations développée dans le cadre 
du stage chez Bewize. L'implémentation a couvert l'ensemble des 
composants de la solution : les technologies utilisées, le formulaire 
WebView React JS, le back-end Spring Boot avec ses APIs REST, la base 
de données PostgreSQL et le dashboard monitor.

Chaque composant a été développé en suivant les bonnes pratiques du 
développement logiciel moderne, avec une attention particulière portée 
à la qualité du code, à la validation des données et à la gestion des 
erreurs. Le chapitre suivant présentera les résultats des tests 
réalisés et une évaluation globale de la solution développée.
